Skittle Sorter & Dispenser

Electromechanical Candy Distribution Unit

For my computer engineering final project in grade 12, I worked in a group of 3 to build an autonomous Skittle sorter and dispenser, making the frame out of LEGO and using Arduino to control the motors and colour sensor. Over a period of several weeks, my team and I assembled the frame, along with power supply, motor attachments and sensor interfacing. Various custom 3D-printed components were created, including a wheel and shroud, through which Skittles would be rotated prior to dispensing.

Motors would control the axial rotation of the machine body, in addition to the allocation wheel. An integrated colour sensor would determine the colour of each incoming Skittle and assign the bin to which the Skittle would be allocated. To optimize accuracy, I compiled a consistency analysis of Skittle readings to determine which RGB component would most reliably correspond to the colour of Skittle.

Multiplayer Pong

Pong with Arduino microcontroller

In a spin on an old classic, I programmed multiplayer Pong on an 8x8 LED matrix with use of an Arduino microcontroller. Switches acted as button-controllers to move paddles for two players.

Buttons could be used to change direction of paddle through rapid clicking, subject to a time threshold. I developed logic tables to determine pin configurations for row-column states. A given combination of ON and OFF for rows and columns would produce a certain LED display, which could be cycled through for animation. I implemented functions in Arduino to toggle the states of rows and columns to program displays more easily.

Divergent RPG

Story-based adventure in C#

A role-playing game where the user chooses a predetermined possibility from a list of given options - resulting in divergent outcomes for acting upon a set of initial conditions. The user, with their attributes (for age, gender, socioeconomic class etc.), will embark on an adventure that progresses according to the choices of the user.

If the character the user is controlling is dead or cannot progress for some other logical reason, the user game ends and statistics are shown. Otherwise, the statistics are shown once the user optionally ends the game.

Coded in C#, Visual Studios 2012. Created 2014, published 2017.

3D Dogfighter

Aerial Combat with Unity

For the Waterloo Game Jam, my team and I developed a multiplayer aerial combat game in C with the Unity game engine. In this game, two or more players can ready-up into an exciting battle in the skies, filled with deadly bullets. To this end, I was responsible for integrating networking with a match-making system for hosting and synchronizing player actions, along with designing fluid combat and flight mechanics, a user interface and terrain. The final product was a reasonably fleshed-out dogfighter, with a fully functional lobby, breathtaking world scenery and uninterrupted bullet mayhem (aside from the occasional death and respawn). Our game was well-received, winning the ”Most Over the Top/Ambitious” award by popular vote. Through this project, I also learned the incredible value of planning for fulfilling project deliverables and to actively overcome problems, of which there were plenty.

Student Database Management System

Secure and easy way to register students

In consultation with the department of physical education at Westmount Secondary School, I was tasked with developing a portal for a new initiative of outdoors programs which were to be released the following year. In partnership with McMaster University Recreation and their Leadership Program, the initiative would help create active, ecologically aware and outdoors-experienced leaders.

This project sought to provide the following administrative benefits:
• Students will have easy and immediately available access to course information and opportunities
• Programs and courses will be organized neatly and logically in a digitally centralized system
• Student information will be clearly organized and privately accessible to all participating students and teachers

To this end, I implemented a database-driven website that integrated each of the overall course expectations with opportunities the initiative would offer, using WAMP stack. Using a wealth of web technologies including CSS, HTML, PHP and JS, along with MySQL for the database, I created a home interface from scratch, with a login-authenticated database for securely adding, viewing and deleting student records. Robust input sanitation with custom error handlers and regular expressions were used to ensure cleaness and validity of database information.
